# 简历工作区使用说明

## 📋 目录用途

本目录用于沉淀简历素材、维护简历草稿，并借助 AI 提炼高质量的简历条目。

---

## 🔄 使用流程

**第一步：填写项目素材（`projects/` 目录）**

打开 `projects/` 下对应的项目文档，以流水账的方式记录：
- 我做了什么
- 遇到了什么难题
- 怎么解决的
- 结果是什么

不需要写得很精炼，越细越好，流水账就行。AI 会帮你提炼。

**第二步：用 AI 把素材转为简历条目**

把项目素材文档的内容发给 AI，使用下方的「素材转简历条目 Prompt」。

**第三步：汇总到 `resume-draft.md`**

把 AI 生成的条目复制到 `resume-draft.md` 对应的项目区块，替换占位符。

**第四步：整体润色**

简历基本成型后，使用下方的「整体润色 Prompt」对整份简历做最终打磨。

---

## 🤖 可复用 Prompt 模板

### Prompt 1：素材转简历条目

```
你是一位擅长写大厂简历的职业顾问，专注于资深工程师岗位。

我将提供一个项目的背景素材，请帮我将其提炼为 3-5 条简历条目。

要求：
1. 每条条目结构为「技术决策/行动 + 方法/手段 + 量化结果」
2. 禁止使用以下模糊动词：负责、参与、熟悉、了解、配合、协助
3. 每条不超过 60 个字
4. 突出架构决策和技术深度，而不是功能罗列
5. 使用主动语态，体现"我主导/我设计/我解决"

以下是项目素材：
[粘贴 projects/ 下的对应文档内容]
```

---

### Prompt 2：整体润色

```
你是一位大厂招聘官，也是资深简历顾问。

我将提供一份前端/全栈工程师的简历草稿，请帮我做整体润色。

要求：
1. 统一全文的表达风格（简洁、主动、量化）
2. 检查并消除重复表达
3. 按以下叙事主线重新排序项目：
   技术广度（多端/工程化）→ 技术深度（架构/性能）→ AI/Agent 方向
4. 控制整体篇幅在一页 A4 以内
5. 标注出哪些条目还不够强，建议如何补充

以下是简历草稿：
[粘贴 resume-draft.md 的内容]
```
